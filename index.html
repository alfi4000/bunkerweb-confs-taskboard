tasks.json Format

Place tasks.json in the same folder as taskboard.html. The board polls it every 30 seconds and automatically adds any new task IDs it hasn't seen before.

{ "version": 1, "project": "My Project", "tasks": [ { "id": "task-001", // unique — used for dedup "title": "Task title", // required "description": "Details…", // optional "priority": "high", // high | medium | low "column": "planned", // planned | next | inprogress // | blocked | done "tags": ["bug", "ui"], // array of tag strings "created": "2026-03-13" // ISO date string } ] }
Dedup logic: if a task ID already exists on the board (moved by you), the board keeps your column assignment and only updates the title/description/priority from the file.

Tip: serve this folder with a local HTTP server (e.g. python3 -m http.server) or deploy to GitHub Pages for live auto-sync to work.
Task Source URL

The taskboard fetches tasks.json from this URL every 30 seconds and auto-imports any new task IDs.

Default (GitHub raw):
https://raw.githubusercontent.com/palmcoasty/bunkerweb-confs-taskboard/refs/heads/main/tasks.json

To use a local file, set ./tasks.json (requires an HTTP server, not file://).
Point to any public raw JSON URL — e.g. your own GitHub repo's tasks.json.
